# ✋ Hand Gesture Recognition AI — Transfer Learning with MobileNetV2

> **SkillCraft Technology | Machine Learning Internship — Task 04**

An AI-powered web application that recognizes **hand gestures from images** using
**transfer learning** with **MobileNetV2**, served through a clean, light-themed
**Gradio** interface. Upload a photo of a hand gesture and the model instantly
predicts which of **10 gesture classes** it belongs to — along with a confidence
percentage.

![App Preview](outputs/app_interface_preview.png)

---

## 📖 Introduction

Hand gesture recognition is a core building block for **touchless human-computer
interaction**, sign-language interpretation, robotics control, gaming, and
accessibility tools. Training a deep convolutional network from scratch for this
task normally requires huge amounts of data and compute.

This project instead uses **transfer learning**: it takes **MobileNetV2**, a
lightweight convolutional neural network pretrained on the massive **ImageNet**
dataset, freezes its learned feature-extraction layers, and attaches a small
custom classification head trained specifically to recognize **10 hand gesture
classes** from the **LeapGestRecog** dataset.

The final model is wrapped in an interactive **Gradio** web app (`main.py`) where
users can upload their own images and get instant predictions with confidence
scores.

## ⚙️ Workflow

The end-to-end pipeline implemented in `hand_gesture_training.ipynb` follows
these steps:

1. **Download the dataset** — fetch LeapGestRecog from Kaggle using `kagglehub`.
2. **Explore the data** — inspect folder structure, count images per class,
   visualize class distribution.
3. **Build a labeled dataframe** — map every image file path to its gesture
   label.
4. **Split the data** — stratified **70% train / 15% validation / 15% test**
   split.
5. **Create data generators** — resize to `224×224`, convert grayscale frames
   to RGB, apply MobileNetV2 preprocessing (`[-1, 1]` scaling), and augment the
   training set (rotation, zoom, shift, brightness).
6. **Build the model** — load MobileNetV2 (ImageNet weights), **freeze** the
   base, and attach a custom dense classification head.
7. **Train** — fit the model with `EarlyStopping`, `ReduceLROnPlateau`, and
   `ModelCheckpoint` callbacks.
8. **Visualize training curves** — plot training/validation accuracy and loss.
9. **Evaluate** — compute test accuracy, classification report, and confusion
   matrix.
10. **Visualize predictions** — display sample test images with predicted
    labels and confidence scores.
11. **Save the model** — export the trained network as `gesture_model.h5`.
12. **Serve via Gradio** — `main.py` loads the saved model and exposes an
    interactive web UI for live predictions.

## 🧠 Transfer Learning Explanation

### Why MobileNetV2?

MobileNetV2 is a **lightweight, efficient CNN architecture** designed for
mobile and embedded vision applications. It uses **depthwise separable
convolutions** and **inverted residual blocks** to achieve strong accuracy
with far fewer parameters than larger networks like VGG16 or ResNet50 — making
it ideal for fast inference in a web app.

### Architecture Used in This Project

```
Input (224 x 224 x 3)
        │
        ▼
┌───────────────────────────────┐
│   MobileNetV2 Base (frozen)    │   <- Pretrained on ImageNet
│   weights = "imagenet"         │      base_model.trainable = False
│   include_top = False          │
└───────────────────────────────┘
        │
        ▼
GlobalAveragePooling2D
        │
        ▼
Dense(256, activation="relu")
        │
        ▼
Dropout(0.3)
        │
        ▼
Dense(128, activation="relu")
        │
        ▼
Dropout(0.2)
        │
        ▼
Dense(10, activation="softmax")   <- Gesture class probabilities
```

### Key Concepts

- **Frozen base layers**: MobileNetV2's convolutional layers were trained on
  1.4M+ ImageNet images and already know how to detect edges, textures,
  shapes, and contours. By freezing `base_model.trainable = False`, we **reuse**
  this knowledge without retraining it — saving time and avoiding overfitting
  on a smaller dataset.
- **Custom classification head**: Only the newly added `Dense` layers are
  trained, learning to map MobileNetV2's generic visual features to the
  **10 specific hand gesture classes**.
- **GlobalAveragePooling2D**: Reduces each feature map to a single value,
  drastically cutting the number of parameters compared to flattening.
- **Dropout layers**: Randomly deactivate neurons during training to reduce
  overfitting.
- **Softmax output**: Produces a probability distribution across all 10
  gesture classes — the highest value becomes the predicted class, and is
  shown as the **confidence percentage** in the app.

## 🚀 Installation & Usage

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/skillcraft_task4_gesture_ai.git
cd skillcraft_task4_gesture_ai
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv venv
source venv/bin/activate      # On Windows: venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Train the model

Open `hand_gesture_training.ipynb` in Jupyter, Google Colab, or Kaggle
Notebooks (GPU runtime recommended) and run all cells. This will:

- Download the LeapGestRecog dataset
- Train the MobileNetV2 transfer-learning model
- Generate all plots in `outputs/`
- Save the trained model to `saved_model/gesture_model.h5`

> Already have a trained `gesture_model.h5`? Just drop it into `saved_model/`
> and skip straight to step 5.

### 5. Launch the web app

```bash
python main.py
```

Then open the local URL printed in the terminal (typically
`http://127.0.0.1:7860`) in your browser.

### 6. Use the app

1. Upload an image of a hand gesture (or click one of the example images).
2. Click **🔍 Predict Gesture**.
3. View the predicted gesture, its confidence percentage, and the full
   confidence breakdown across all 10 classes.

## 🔮 Future Scope

- **Real-time webcam inference** — extend the Gradio app to classify gestures
  live from a webcam feed.
- **Fine-tuning the backbone** — unfreeze the top layers of MobileNetV2 and
  fine-tune with a low learning rate for further accuracy gains.
- **Larger gesture vocabulary** — extend beyond the 10 LeapGestRecog classes
  to support sign-language alphabets or custom gesture sets.
- **Mobile deployment** — convert the model to **TensorFlow Lite** for
  on-device inference on Android/iOS.
- **Multi-hand detection** — combine with an object-detection model to locate
  and classify multiple hands in a single frame.
- **Gesture-to-action mapping** — connect predictions to real actions, such as
  controlling media playback, smart-home devices, or presentation slides.
- **Robustness improvements** — train with additional augmentation (varying
  lighting, backgrounds, and skin tones) to improve generalization to
  real-world webcam images, since LeapGestRecog images are captured under
  controlled infrared conditions.

## 💡 Key Learnings

- Applied **transfer learning** by freezing a pretrained MobileNetV2 backbone
  and training only a custom classification head.
- Built an image preprocessing pipeline that converts grayscale near-infrared
  images to the RGB format expected by ImageNet-pretrained models.
- Used `ImageDataGenerator` for **on-the-fly data augmentation** to improve
  generalization.
- Evaluated a multi-class image classifier using accuracy, a classification
  report, and a confusion matrix heatmap.
- Deployed a trained Keras model behind an interactive **Gradio** interface
  with example images, confidence scores, and a custom light theme.
